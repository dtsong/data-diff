# Governance

## Overview

data-diff is a community-maintained open-source project. It was originally created by Datafold, who sunset the project in May 2024. The community has since adopted and continues to develop it.

## Contributing

This project is open to all contributors. Whether you are fixing a typo, adding a database driver, or proposing a major change, your contributions are welcome. See [CONTRIBUTING.md](CONTRIBUTING.md) for how to get started.

## Decision Making

- **Pull requests** are reviewed by maintainers. At least one maintainer approval is required before merging.
- **Issues and feature requests** are triaged by maintainers. Community input (upvotes, comments) helps prioritize work.
- **Major changes** (breaking API changes, new architecture, governance updates) are discussed in a GitHub issue or discussion before implementation.

## Maintainers

Maintainers are contributors who have demonstrated sustained, quality contributions and have been granted merge access. New maintainers are nominated by existing maintainers.

## Code of Conduct

Be respectful and constructive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).
